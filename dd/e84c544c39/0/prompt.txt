# AiCuty 常設ページ + イベントシステム (`/e/`)

## Context

Bottom nav の「Create」は終了済みイベント (Voice Audition) を指している。AiCuty を常設コンテンツとして独立ページ化し、イベントは `/e/` 配下にショートコード管理で SEO 対応する。Wix のイベント一覧から段階的に移植していく基盤を作る。

**状態管理**: イベントは `preview` → `live` → `archive` と遷移。live は既存の静的 HTML (`public/e/Fes26Halu/`)、archive は React 内で描画。

---

## ルーティング設計

| Path | Handler | 説明 |
|:---|:---|:---|
| `/aicuty` | React (Layout) | AiCuty 常設ページ (bottom nav) |
| `/e` | React (Layout) | イベント一覧 |
| `/e/Fes26Halu` | 静的 HTML (CF Pages `_redirects`) | ライブイベント |
| `/e/VoiceAudition` | React → EventArchive → CreateVoice | アーカイブ |
| `/e/<slug>` | React → EventArchive | 汎用アーカイブ |
| `/create/voice` | React → CreateVoice | 後方互換 (維持) |

**SW / _redirects の共存**: 静的 HTML イベントは `_redirects` で先にマッチ。残りの `/e/*` は SPA fallback。`navigateFallbackDenylist` は静的ページのみに限定。

## イベント状態遷移の運用

イベントが `live` → `archive` に変わるとき:

1. `events.js` の `status` を `'archive'` に変更
2. `_redirects` から静的ページルールを削除
3. `vite.config.js` の denylist から該当パスを削除
4. (任意) `public/e/<shortcode>/` を削除 or 保持

逆に新イベントを `live` にするとき:

1. `public/e/<Shortcode>/index.html` を作成
2. `_redirects` に静的ルールを追加
3. `vite.config.js` の denylist にパスを追加
4. `events.js` に `status: 'live'` で追加

     ```js
     import { useEffect } from 'react';
     const SITE_NAME = 'AICU Japan';
     export default function usePageMeta(title, description) {
       useEffect(() => {
         const prev = document.title;
         document.title = title ? `${title} | ${SITE_NAME}` : `${SITE_NAME} — クリエイティブAI時代に つくる人をつくる`;
         let metaDesc = document.querySelector('meta[name="description"]');
         const prevDesc = metaDesc?.getAttribute('content');
         if (description && metaDesc) {
           metaDesc.setAttribute('content', description);
         }
         return () => {
           document.title = prev;
           if (prevDesc && metaDesc) metaDesc.setAttribute('content', prevDesc);
         };
       }, [title, description]);
     }
     ```
